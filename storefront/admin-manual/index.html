Online StoreFront Admin Instruction Manual

Internal guide for store admins handling orders, promos, tickets, Lalamove, customer updates, and admin-only actions in the JCIT Online StoreFront with AI Concierge system.

Quick Access

Store: https://poppers.jcit.digital/ Admin: https://poppers.jcit.digital/admin Tracking: https://poppers.jcit.digital/poppers/track Telegram-only action: Contact Customer

Daily Workflow

1. Log in to Admin

Use your admin username and passcode in the admin portal.

2. Review Orders

Check pending, awaiting payment, and Lalamove-ready orders first.

3. Verify or Update

Confirm payment, book delivery, upload order photos, and move statuses forward.

4. Monitor Support

Resolve tickets, keep promos current, and review customer surveys regularly.

Orders Tab

Statuses You Can Use

  • Pending Confirmation
  • Awaiting Payment Verification
  • Confirmed
  • Preparing
  • Out for Delivery
  • Delivered
  • Completed
  • Cancelled

Order Actions

  • Save Update: changes status and tracking details
  • Verify Payment: confirms prepaid orders and can auto-book Lalamove
  • Book Lalamove: manually trigger Lalamove booking
  • Send Tracking Link: works globally; browser orders use the tracking page
  • Cancel Order: marks the order cancelled
  • Upload Order Photos: can upload multiple photos that show on tracking

Contact Customer

This is the only Telegram-only admin action. It works only when the order has a real Telegram chat ID. Browser orders should be handled through the saved contact number and tracking page instead.

Payments and Delivery

Prepaid Orders

Verify payment first. For same-day Lalamove orders, the delivery fee is already included in the final bill. The customer should not pay the rider separately.

Cash on Delivery

COD orders include the COD fee in the bill. Only COD remains payable on delivery.

Pickup Points

When booking Lalamove, choose the correct pickup point in admin before submitting: Jay Concha or Josh at Vine Residences.

Promos, Rewards, and Referrals

Promo Rules

Only one promo code can be used per order. Promo codes cannot stack.

Rewards Rules

Rewards and promo codes are not stackable. Customers can redeem PHP 100 for every 1000 points.

Repeat Buyer Logic

Repeat buyers can auto-receive LOYAL30 when no other promo is entered.

Referral Credit

Referral points are awarded after the referred order is completed. Review promo and rewards behavior before making manual adjustments.

Other Admin Tabs

Tickets

Open, close, and monitor support, bulk-order, and related service requests.

Promos

Create or update promo codes, set discounts, and toggle active or inactive status.

Inventory

View and update stock, price, and active status. Inventory should stay admin-only.

Admins

Add admin users, reset passcodes, and manage internal access. Telegram-native 2FA is not available via Bot API.

Surveys

Review customer ratings, comments, and the average satisfaction score for completed orders.

Admin Notes and Best Practices

Browser Orders

Browser orders may not have Telegram chat IDs. Use saved contact details and tracking links instead.

Telegram Orders

These can receive direct Telegram admin messages when Contact Customer is available.

Admin Notifications

All admin actions should send a Telegram admin notification so the team has an audit trail.
